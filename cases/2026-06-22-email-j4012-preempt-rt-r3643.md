## 问题摘要

英文客户 Akash Potti（Pace Robotics）在 **reComputer Industrial J4012** 上请求 PREEMPT_RT 内核详细启用步骤。当前系统：Ubuntu 22.04，内核 `5.15.148-tegra`，L4T R36.4.3 / JetPack 6.2。此前 Seeed 已建议不要使用 NVIDIA 标准 RT kernel APT 包，改用 Seeed 定制 BSP。

客户具体问题：
- 应用哪个 BSP source tag
- PREEMPT_RT 是否已包含在 BSP 源中
- 下载/打补丁/编译内核与模块/DTB 的具体命令
- 能否在运行系统上安装 RT 内核，还是必须整系统 reflash
- 是否有 J4012 + JP 6.2 / R36.4.3 专用指南

## 答复要点

- **BSP tag**：`Seeed-Studio/Linux_for_Tegra` 分支 `r36.4.3`；NVIDIA `source_sync.sh -t jetson_36.4.3`。
- **PREEMPT_RT**：未预置；通过 `generic_rt_build.sh enable` 或 Seeed `nvbuild.sh -r` 启用。
- **构建**：按 Seeed BSP 构建 Wiki，在 `apply_binaries.sh` 之后用 `./nvbuild.sh -r` 编译，`./do_copy.sh` + `./nvbuild.sh -r -i` 安装到 rootfs，`l4t_update_initrd.sh` 更新 initrd。
- **部署**：推荐 host 编译后 `l4t_initrd_flash.sh` 刷机；**不要**使用 NVIDIA `nvidia-l4t-rt-kernel` APT 包。
- **专用指南**：无 J4012 专用 RT 文档；参考 Seeed BSP Wiki + NVIDIA Kernel Customization RT 章节 + Forum A603 社区帖。
- **已确认产品形态**：reComputer Industrial J4012；flash device name 为 `recomputer-industrial-orin-j201`。
- **待确认**：OS 安装在 NVMe 还是 eMMC；是否必须保留用户数据。
