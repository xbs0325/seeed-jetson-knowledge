# 用户偏好与渠道约定

> 由对话沉淀；有变更时通过 PR 更新。

## 场景默认

| 触发 | 默认行为 |
| --- | --- |
| 用户说「邮件」 | 按 **Zoho** 英文技术邮件流程处理 |
| 中文问题（非邮件） | **快速检索** `INDEX.md` + `docs/`，**简短直接**回答 |
| 淘宝/天猫买家问题 | 更短、可复制到聊天窗口 |

## 回复节奏

- 英文邮件：先内部建议（翻译、总结、依据、待确认），**用户确认后再写外发英文**。
- **外发英文邮件要简短**：直击结论与必要说明，避免长段落堆砌；能一句说清的不写三句。
- 生成英文外发回复时，英文正文结束后附上对应的**完整中文翻译**，方便内部核对和复制。
- 中文问题：不走完整邮件流程，优先速度。

## 仓库协作

- 优先通过 GitHub 远端连接器读取和修改仓库；本地工作区不可用时，以远端仓库内容为准。
- 当前阶段：改动走 **PR**，不直推 `main`。
- 指令与知识库暂放同一分支体系，不拆多分支。

## 记忆沉淀

- 涉及**记忆**的内容（偏好、约定、习惯），Agent 应写入 `memory/` 对应文件并开 PR，而非仅留在对话里。
- 用户说「**以后也这样**」「**默认……**」「**记住……**」等长期约定时，**当场主动更新** `memory/` 或 `instructions/`，不要等用户再提醒。
